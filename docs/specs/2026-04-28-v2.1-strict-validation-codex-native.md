## 1. Goals

1. **Strict literature cross-validator** — DOI is the hard gate; nothing without a verified DOI counts toward `selected_count`. Two enrichment passes (OpenAlex/Semantic Scholar + Consensus topic-level), one optional claim-support pass.
2. **Iterative plotter** — 3-cycle refine loop, two output paths (Python scripting + LaTeX-native via TikZ/pgfplots), publication-grade defaults, automated VLM critique per cycle.
3. **Human-attitude writer** — anti-LLMish blacklist enforced; critical/skeptical tone; intra-phase ideation re-dispatch when claims are unquantified or unsupported.
4. **Codex-primary dispatch** — `spawn_agent` is the canonical surface; Claude Code and Gemini are parity peers. Background workflow stays identical from the user's view.

## 2. Non-goals (v2.1.0)

- No model-stack changes (still Opus 4.7 / GPT-5.5 / Gemini 3.1 Pro per host).
- No new Sakana modules vendored beyond what landed in v2.0.0.
- No paid Consensus / Scite Enterprise tiers required to run the pipeline (graceful Free/Pro fallback).
- No automatic Anna's Archive download for closed-access papers (OA-only, gated by `oa_status`).

## 4. Strict literature cross-validator (Goal 1)

### 4.1 Stage 1 — DOI gate (hard)

```python
def stage1_doi_gate(paper, harvest_title) -> dict:
    # Syntactic regex (DOI Foundation grammar)
    if not re.match(r"10\.\d{4,9}/[-._;()/:A-Za-z0-9]+", paper.get("doi","")):
        return {"passed": False, "reason": "no_doi"}
    # Crossref polite-pool resolve (10 req/s, 3 concurrent)
    r = httpx.get(f"https://api.crossref.org/works/{doi}?mailto={EMAIL}")
    if r.status_code != 200:
        agency = httpx.get(f"https://api.crossref.org/works/{doi}/agency").json()
        if agency.get("agency", {}).get("id") == "datacite":
            r = httpx.get(f"https://api.datacite.org/dois/{doi}")
            if r.status_code != 200:
                return {"passed": False, "reason": "doi_404_both_registries"}
    # Title fuzzy match — token_sort_ratio ≥ 0.85 (drop on mismatch)
    score = rapidfuzz.fuzz.token_sort_ratio(harvest_title, registry_title) / 100
    if score < 0.85:
        return {"passed": False, "reason": f"title_mismatch_{score:.2f}"}
    return {"passed": True, "doi": doi, "title_score": score}
```

Failures land in `references_validation.json` under `dropped_papers[]` with reason — never silently dropped.

### 4.2 Stage 2 — Enrichment cascade (soft)

For papers passing Stage 1 with missing optional metadata, cascade:

1. **OpenAlex** (free with API key since Feb 13, 2026; singleton calls cost 0 credits) — fills title, authors, year, venue, abstract (inverted-index reconstruction), OA URL.
2. **Semantic Scholar bulk POST** (`/paper/batch`, up to 500 DOIs / 1 call at 1 req/s with key) — fills abstract, OA PDF URL, `tldr` (used in Stage 3).
3. **Consensus MCP** (`search` tool, topic query) — Free 30/mo, Pro 1,000/mo. Used for **topic-level corroboration only** because DOI return is Enterprise-only. Validates that the paper appears in independent semantic search for the topic.
4. **Anna's Archive** (`mcp__annas-mcp__article_search` by DOI) — gated by `oa_status == "open"`; never run on closed-access papers. Fills missing abstract from OA full-text snippet.
5. **PubMed E-utilities** — biomedical-only; adds MeSH terms when `is_biomedical(paper)` is true.

Each enrichment writes its source to `paper["enriched_from"]` so later citation copy can attribute.

### 4.3 Stage 3 — Claim support (optional, top-cited only)

For papers cited ≥3 times in the manuscript:

```python
def stage3_claim_support(paper, claim) -> dict:
    tldr = paper.get("s2_tldr","") or paper.get("abstract","")[:500]
    score = rapidfuzz.fuzz.partial_ratio(claim.lower(), tldr.lower())
    return {"checked": True, "method": "s2_tldr_or_abstract",
            "match_score": score, "flag": score < 40}
```

Flagged claims surface in `claim_support_matrix.md` for human review. Never used as a hard gate.

### 4.4 Throughput target

For 200 papers, asyncio + `httpx.AsyncClient`:
- Stage 1 (Crossref polite, 3 concurrent): **~40 s**
- Stage 2 (OpenAlex 0-credit singletons + S2 bulk POST): **~5 s**
- Stage 3 (top-20 only, S2 tldr already cached): **~0 additional s**
- **Total wall-clock: ~1 minute for 200 papers**

## 5. Iterative plotter (Goal 2)

### 5.1 3-cycle structure

| Cycle | Input | Action | Output |
|---|---|---|---|
| **1 — Inspect & draft** | `results.csv` + `*.npy` | Schema sniff → plot-type rationale → first PNG draft at 2× DPI | `figures_draft1/` + `manifest.json` (per-figure type rationale) |
| **2 — VLM critique & refine** | Draft PNGs + captions | VLM 10-point rubric (legibility, color-blind safety, axis labels, error bars, caption-claim match, data-ink ratio…) per figure | `figures_draft2/` + `vlm_rubric.json` |
| **3 — Polish & export** | Refined PNGs + journal style | Apply `rcParams` (Okabe-Ito palette, journal column-width, font embedding) → vector PDF + 300dpi TIFF | `figures_final/` + `figure_metadata.csv` + `figures.tex` |

Cycle 1 and 3 process figures in parallel. Cycle 2 is sequential per figure (VLM context-window limit).

### 5.2 Two output paths

**Scripting (matplotlib/seaborn):**
- Default: Okabe-Ito categorical palette, Wong fallback for n>8
- Vector output (PDF/EPS) for line/scatter; TIFF 300dpi for heatmaps
- Per-journal `rcParams` profiles (Nature/Cell/IEEE/Springer)
- `proplot` available for users who opt in via setting

**LaTeX-native (TikZ / pgfplots):**
- Auto-generated `.tex` snippets with `\begin{tikzpicture}...\end{tikzpicture}`
- Externalize via `\usetikzlibrary{external}` → cached PDFs (cuts compile time)
- `pgfplotstable` for booktabs-styled summary tables (review articles)
- `pgf-pie` for category breakdowns
- Falls back to `\includegraphics{figure.pdf}` if `--shell-escape` unavailable

### 5.3 Article-type-specific defaults

- **Review article** → bibliometric figures (timeline, taxonomy, co-citation network); booktabs summary tables; LaTeX-native preferred for font consistency.
- **Experimental** → headline result + ablation table + Pareto front; matplotlib scripting preferred (data-driven aspect).
- **Benchmark** → score-vs-method bars w/ 95% bootstrap CI, per-task heatmap, compute-vs-quality scatter; Bradley-Terry Elo bars w/ symmetric CI.

### 5.4 Per-cycle CLI

```bash
mcp__ai-scientist__run_plotter_cycle --cycle 1 --figures all
mcp__ai-scientist__run_plotter_cycle --cycle 2 --vlm gpt-4o
mcp__ai-scientist__run_plotter_cycle --cycle 3 --journal nature
```

Each cycle is independently re-runnable.

## 6. Human-attitude writer (Goal 3)

### 6.1 Anti-LLMish lint

`anti_llm_lint.py` runs after `phase_5_manuscript` and before `phase_6_citations`. Blacklist is empirically grounded — not a stylistic opinion but a measured corpus tell from Liang et al. 2024 (15.1M PubMed abstracts) and the ICLR 2024 peer-review excess-vocabulary study.

**Tier 1 — measured frequency ratios in post-ChatGPT corpora (block on any single occurrence):**

| Word | Excess ratio | Source |
|---|---|---|
| `meticulous(ly)` | 34.7× | ICLR 2024; Gray 2023 |
| `delve(s) / delving / delved` | 28.0× | PubMed 15M-abstract study |
| `underscore(s) / underscoring` | 13.8× | PubMed 15M-abstract study |
| `intricate / intricacies` | 11.2× | ICLR 2024 |
| `showcas(e/ing/ed)` | 10.7× | PubMed 15M-abstract study |
| `commendable` | 9.8× | ICLR 2024 |
| `crucial` | 9.8× | ICLR 2024 + PubMed |
| `pivotal / realm` | top-ranked | Liang et al. |

**Tier 2 — strong signal across editorial + community sources (warn on ≥2 in a section):**

`robust`, `landscape`, `vibrant`, `tapestry`, `testament`, `leverages`, `harness`, `foster(ing)`, `navigate`, `embark`, `paradigm`, `multifaceted`, `nuanced`, `transformative`, `revolutionary`, `comprehensive`, `holistic`, `seamless`, `bustling`, `treasure trove`, `wealth of`, `plethora`, `myriad`, `unveil`, `unleash`, `cutting-edge`, `state-of-the-art` (legal only when citing a benchmark score, not as adjective), `unprecedented`, `groundbreaking`, `renowned`, `boasts`, `innovative`, `garner`, `enduring`, `interplay`, `align(s) with`, `highlighting`, `emphasizing`, `bolstered`, `enhance(d)`.

**Tier 3 — phrase patterns (regex match, block):**

- Epistemic-authority surrogates: `studies have shown`, `research indicates`, `experts agree`, `it is widely accepted`, `it is well established`, `evidence suggests` (without immediate citation)
- Structural fillers: `it is important to note`, `it should be noted`, `it is worth mentioning`, `in conclusion`, `ultimately,`, `while it is true that`, `in the realm of`, `in the world of`, `in the era of`, `plays a (crucial|pivotal|key) role`
- Paragraph-initial transitions: `^(Furthermore|Moreover|Additionally|Notably|Importantly|Interestingly|Remarkably|Fascinatingly),`
- Sycophantic openings: `^(Certainly|Absolutely|Of course),`

**Tier 4 — style metrics (rate-based, not single-word):**

- **Em-dash density**: warn if > 2 per 1,000 words (human academic rate); block if > 5 per 1,000 words (GPT-4.1 averages 10.6, Claude Opus 4.6 averages 9.1, human < 2)
- **Tricolon adjective stacks**: regex `(\w+, \w+,? and \w+)` flagged when all three are evaluative adjectives (e.g., `robust, scalable, and efficient`) — each must have a metric or be cut
- **Participial commentary clauses**: `, (highlighting|underscoring|demonstrating|emphasizing) [^.]+\.` flagged
- **Elegant variation (synonym-rotation)**: when the same concept is referred to with > 2 different surface forms within a paragraph (`the approach / the framework / the methodology`)
- **Copulative-verb avoidance**: warn if `is/are/was/were` density < 5% of sentences in a section (LLMs substitute `serves as / stands as / marks / features`)

**Disciplinary exemptions:**

- `crucial` allowed when describing a rate-limiting biochemical step ("the crucial Ser473 phosphorylation")
- `robust` allowed in `robust regression / robust stability` technical sense
- `comprehensive` allowed in systematic-review methods sections
- `novel` allowed when bounded by an explicit prior-work survey citation
- `state-of-the-art` allowed when accompanied by a benchmark score reference

These exemptions are encoded as context-aware rules (regex with disambiguating co-occurrence terms), not blanket whitelisting.

**Output:** `anti_llm_lint.json` with per-occurrence line/column, tier, term, suggested rewrite. Manuscript writer dispatches with this JSON inlined in `<input name="prior_attempts">` for the next reflection round, max 3 rounds.

### 6.2 Intra-phase ideation re-dispatch

A separate `claim_audit.py` module scans for the 14 unquantified-claim patterns (per anti-LLMish research §D):

| Pattern | Trigger condition |
|---|---|
| `outperforms / better than / improves` | no `(p\s*[<=]|\d+%|n=\d+|[\d.]+\s*\b(ms\|s\|GB\|MB)\b)` within 200 chars |
| `novel / first to` | no nearby `(see Appendix|search protocol|to our knowledge)` + no citation |
| `scalable` | no complexity analysis (`O\(`) or empirical curve mention |
| `efficient` | no runtime/memory comparison numbers |
| `robust` | no `(adversarial\|distribution shift\|noise (level\|model)\|SNR)` mention |
| `generalizes` | no `(held-out\|OOD\|out[\s-]?of[\s-]?distribution\|test set)` mention |
| Mechanistic claim (e.g. `attention allows / mechanism enables`) | no nearby `(ablation\|lesion\|probe\|removed)` study reference |
| User-preference claim | no `(n=\d+|\d+ participants\|survey\|interview)` |
| `significant` | no `(p\s*[<=]|F\(|t\(|χ²\|chi-square)` test stat |
| Causal verb on observational data | section is described as observational + verb is `cause(s/d)\|leads to\|results in` |
| `applicability / generalizability` to other domain | no test in target domain |
| Missing negative results | section labeled `Results` with 100% positive findings (heuristic) |
| Comparison to non-contemporaneous baseline | named baseline > 5 years before paper year |

The scanner emits a `clarification_request` and the orchestrator dispatches a single ideation cycle scoped to that paragraph:

```python
def intraphase_ideation_redispatch(paragraph, claim_pattern):
    return {
        "agent_name": "ideator",
        "inputs": {
            "mode": "clarify_claim",
            "paragraph": paragraph,
            "vague_pattern": claim_pattern,
            "instruction": "Replace the vague claim with a specific quantification or hedge it appropriately."
        }
    }
```

Re-dispatch is bounded at max 3 paragraphs per draft to prevent runaway loops.

### 6.3 Critical-tone enforcement

Section-level checklist applied during `phase_5_manuscript`:

- Every empirical claim has a citation OR an explicit hedge (`appears`, `suggests`, `is consistent with`)
- No first-person superlatives about own work (`our novel approach`, `we demonstrate definitively`)
- Limitations section is required (auto-detected: heading regex)
- Methods/Search-strategy section required for review articles
- Negative-results paragraph encouraged (warning, not failure, if absent)

## 7. Codex-primary dispatch (Goal 4)

### 7.1 `dispatch/codex_native.py` design

Replaces the v2.0.0 stub `codex.py` with real `spawn_agent` integration:

```python
class CodexNativeDispatcher:
    def __init__(self, *, spawn_agent: Callable, wait: Callable,
                 close_agent: Callable, max_threads: int = 6):
        self.spawn_agent = spawn_agent
        self.wait = wait
        self.close_agent = close_agent
        self.max_threads = max_threads

    def dispatch_wave(self, *, agent_name: str, inputs_list: List[dict]) -> List[dict]:
        """Fan-out / fan-in with slot-leak guard for issue #18335."""
        spawned = []
        for inputs in inputs_list[: self.max_threads]:
            r = self.spawn_agent(
                message=self._build_message(agent_name, inputs),
                agent_type="worker",
                model="gpt-5.5",
                reasoning_effort="xhigh",
                fork_context=False,
            )
            spawned.append(r["agent_id"])
        # wait for all
        statuses = self.wait(ids=spawned, timeout_ms=600_000)
        # ALWAYS close — slot leak guard (GitHub issue #18335)
        for agent_id in spawned:
            try: self.close_agent(agent_id)
            except Exception: pass
        return [statuses["status"][aid].get("payload", {}) for aid in spawned]
```
